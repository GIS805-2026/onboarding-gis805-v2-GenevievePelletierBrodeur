# Guides de studio (lab PDFs)

Un PDF par séance. Gardez-le ouvert sur votre machine pendant le sprint.

Chaque guide contient, sur 7 pages :

1. **Couverture** — question du CEO + contexte du soir
2. **Objectifs d'apprentissage** — outcomes, points clés, idées reçues, priorités (core/stretch/preview)
3. **Déroulement** — rythme de la soirée, structure par partie, interactions Wooclap
4. **Lab / Studio** — objectif, livrable attendu, artéfacts du repo, checklist, matériaux
5. **Remise & évaluation** — artéfacts requis, règle de remise, rubric, exit ticket
6. **Outils & règles IA** — toolchain, politique IA, politique cloud
7. **Lectures recommandées**

Les séances d'examen (S05, S10, S14) ont 4 pages (couverture, déroulement, remise, outils).

## Fraîcheur

Ces PDFs sont regénérés automatiquement à chaque mise à jour du cours par
un GitHub Action côté ProfOS. Le fichier date sur GitHub reflète la dernière
régénération.

## Si vous préférez une version en ligne

L'application ProfOS propose un onglet **Lab** qui affiche le PDF dans le
navigateur. Demandez à votre instructeur l'URL si ce n'est pas déjà sur le
`README.md` principal du dépôt.
